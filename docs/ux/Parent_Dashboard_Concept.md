# Parent Dashboard Concept

## Dashboard goal

Help the parent configure the child phone experience without technical confusion.

## Main sections

### 1. Child Home Screen

Purpose:

- preview child home
- assign tiles
- reorder visible tiles
- reset layout

MVP controls:

- add/remove tile
- choose app for tile
- preview layout

### 2. Allowed Apps

Purpose:

- select visible apps
- hide distracting apps from launcher
- assign apps to modes

MVP controls:

- app list
- search/filter
- category labels
- visible/hidden toggle
- mode assignment

### 3. Modes

Purpose:

- configure what appears in each routine mode

MVP modes:

- Home
- School
- Sleep

Later modes:

- Travel
- Exam

Controls:

- set active mode
- choose apps per mode
- explain mode purpose

### 4. Contacts and Emergency

Purpose:

- define parent and emergency contact access

MVP controls:

- parent name
- parent phone number
- emergency contact label
- emergency phone number
- test call action

### 5. Parent Lock

Purpose:

- keep child from editing setup

Controls:

- set PIN
- change PIN
- lock layout editing
- explain PIN recovery limits

### 6. Setup and Limitations

Purpose:

- help parent set default launcher
- explain what the app can and cannot control

Controls/content:

- default launcher guidance
- Android limitation notice
- Family Link recommendation
- troubleshooting

## Dashboard tone

Use plain language.

Example:

Good: “Choose which apps your child can see on the home screen.”

Avoid: “Configure app-level access restrictions.”
