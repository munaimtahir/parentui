# UX Concept

## UX principle

The app has two emotional modes:

1. Child mode: calm, simple, friendly.
2. Parent mode: structured, trustworthy, controlled.

## Child UI direction

Child UI should feel:

- safe
- simple
- clear
- calm
- age-appropriate
- readable

Avoid:

- excessive animation
- tiny text
- hidden gestures
- toy-like design for all ages
- clutter
- confusing app categories
- scary restriction language

## Parent UI direction

Parent UI should feel:

- organized
- adult
- calm
- transparent
- reliable

Avoid:

- playful child-like dashboard
- too many technical terms
- complicated device-management language
- surveillance framing

## Child home screen concept

The child sees:

- time/date
- active mode label
- parent contact tile
- emergency tile
- approved app tiles
- simple mode-specific background or header

## Parent dashboard concept

The parent sees:

- child home preview
- visible apps
- mode settings
- contacts and emergency setup
- parent lock
- setup and limitations

## Visual tone

Recommended:

- soft rounded cards
- clear icon labels
- moderate spacing
- accessible font sizes
- gentle colors
- no aggressive red warning UI except real emergency context

## Interaction rules

- Child should not need gestures.
- Parent setup should be step-by-step.
- Important settings must explain consequences.
- Any Android limitation should be explained in plain English.
- PIN-protected actions should never look like child actions.
