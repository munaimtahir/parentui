# Risks and Assumptions

## Main product risk

Parents may assume the app fully locks Android.

Mitigation:

- explain limitations during onboarding
- avoid “cannot be bypassed” language
- include Family Link recommendation
- keep Play Store wording honest

## Compliance risk

The app involves children as users.

Mitigation:

- no ads at launch
- no analytics at launch
- no child account
- no location tracking
- no unnecessary permissions
- maintain Data Safety accuracy
- maintain privacy policy before store release

## Technical risk

Android launchers cannot control every system path on every device.

Mitigation:

- focus on launcher home experience
- hide apps only inside launcher
- do not claim full system control
- test on multiple Android versions and OEM devices

## UX risk

Parent setup may become too complex.

Mitigation:

- start with 3 modes only
- use presets later
- keep app selection clear
- show child home preview

## Monetization risk

Parents may resist subscription.

Mitigation:

- use free MVP + one-time premium unlock
- avoid ads
- make premium features practical

## Device fragmentation risk

Different OEMs handle default launchers and battery settings differently.

Mitigation:

- document tested devices
- include device-specific help gradually
- avoid features requiring unstable hacks

## Core assumptions

- Parent will set app as default launcher.
- Child uses a dedicated or shared Android phone.
- Parent wants less distraction, not secret surveillance.
- Offline-first is preferred for trust and compliance.
- Google Family Link can be recommended for deeper restrictions.
