# MVP Scope

## MVP goal

Build a practical child-safe launcher that lets a parent set up a simple, controlled home screen for a child’s phone.

## MVP value statement

The MVP should prove that parents can quickly turn a normal Android phone into a calmer, safer, child-facing phone surface.

## MVP must include

### 1. Default launcher support

- App can act as an Android home launcher.
- Parent receives clear guidance to set it as default.
- App does not overpromise system-level lockdown.

### 2. Child home screen

- Fixed tile layout.
- Large readable icons and labels.
- No clutter.
- No exposed full app drawer in child view.
- Child-friendly but not cartoonish.
- Works for younger school-age children and pre-teens.

### 3. Parent PIN

- Parent/admin area protected by PIN.
- Child cannot edit layout without PIN.
- PIN setup during onboarding.
- PIN change option in parent dashboard.

### 4. Allowed apps

- Parent selects apps visible in child launcher.
- Hidden apps are hidden from this launcher only.
- App list should be grouped clearly.
- Parent can assign apps to modes.

### 5. Parent contact tile

- One-tap call to parent/guardian.
- Use manually entered number where possible.
- Avoid unnecessary contact permission.

### 6. Emergency tile

- Emergency/safety contact access.
- Must be visible in child modes unless parent disables it intentionally.
- Use simple language.

### 7. Manual modes

MVP includes three manual modes:

- Home Mode
- School Mode
- Sleep Mode

Travel Mode and Exam Mode are reserved for later.

### 8. Offline-first

- No account required.
- No backend required.
- No cloud sync required.
- All settings stored locally.
- No ads.
- No analytics at launch.

## MVP should not include

- location tracking
- message monitoring
- call recording
- microphone monitoring
- remote dashboard
- browser history tracking
- web filtering
- cloud child profile
- social feed
- AI assistant for child
- ads or ad SDKs
- full device-owner mode
- enterprise kiosk claims
- “cannot be bypassed” claims

## MVP success criteria

The MVP is successful if a parent can:

1. Install the app.
2. Set it as the default launcher.
3. Create a parent PIN.
4. Select approved apps.
5. Add parent/emergency contact.
6. Choose Home, School, or Sleep mode.
7. Give the phone to a child with a clean and predictable home screen.
