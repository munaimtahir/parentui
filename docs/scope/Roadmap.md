# Roadmap

## Phase 1 — MVP Launcher

Goal: prove the core parent value.

Includes:

- launcher replacement
- child home screen
- onboarding
- parent PIN
- allowed app selection
- parent contact
- emergency contact
- Home / School / Sleep mode
- local storage only
- no backend
- no ads
- no analytics

Exit criteria:

- parent can complete setup
- child home screen works as default launcher
- app launches selected apps
- parent dashboard is protected
- modes visibly change app availability
- tests pass

## Phase 2 — Safety and Setup Polish

Goal: make the product usable by real parents.

Includes:

- improved setup wizard
- layout preview
- better app picker
- mode-specific app assignment
- reset layout
- local backup/restore
- Family Link guidance
- improved PIN recovery guidance

Exit criteria:

- onboarding is simple
- parent can understand Android limitations
- app remains privacy-safe
- manual testing confirms setup flow

## Phase 3 — Routine Pack

Goal: create daily family routine value.

Includes:

- scheduled mode switching
- bedtime mode
- exam mode
- travel mode
- optional reminders
- quiet visual design for sleep/study

Exit criteria:

- scheduled modes are predictable
- parent can override manually
- child screen remains simple

## Phase 4 — Managed Device Track

Goal: explore stronger restriction models separately.

Possible future work:

- Android device-owner mode
- school/kiosk edition
- managed deployment
- stricter app restrictions
- enterprise policy documentation

Important:

This track must not change the consumer MVP promise.
